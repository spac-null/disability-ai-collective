Accessibility Statement

Accessibility isn't an afterthought—it's fundamental to who we are and how we build technology.

Our Commitment

Accessibility-First Design

As an experimental arts platform led by disability culture, accessibility is built into every aspect of our work. We don't retrofit accessibility—we design with disabled users as primary stakeholders from day one.

Community Accountability

We're accountable to disabled communities first. Our accessibility practices are reviewed by disabled users who represent diverse disabilities, assistive technologies, and cultural perspectives.

Continuous Improvement

Digital accessibility is an ongoing practice, not a checklist. We regularly test with real users, update our practices based on feedback, and stay current with evolving accessibility standards.

Open Source Accessibility

Our accessibility patterns and code are open source. We share our methods so other organizations can benefit from disability-led design practices.

WCAG 2.1 AA Compliance

We meet or exceed WCAG 2.1 Level AA standards with ongoing testing and community verification.

✅ Perceivable

  • 4.5:1 color contrast minimum (we exceed this)
  • Text can be resized to 200% without losing functionality
  • All images have descriptive alt text
  • Video content includes captions and transcripts
  • High contrast and dark mode available

✅ Operable

  • All functionality available via keyboard
  • No content causes seizures (no flashing)
  • Skip links for efficient navigation
  • 48px minimum touch targets on mobile
  • Respects reduced motion preferences

✅ Understandable

  • Clear, simple language throughout
  • Consistent navigation and layout
  • Error messages are descriptive
  • Context changes are predictable
  • Reading level appropriate for content

✅ Robust

  • Valid, semantic HTML structure
  • Compatible with assistive technologies
  • Proper ARIA labels and landmarks
  • Works across browsers and devices
  • Degrades gracefully without JavaScript

🎯 Beyond AA

  • Dyslexia-friendly font options
  • Cognitive accessibility features
  • Multiple input methods supported
  • Personalization options
  • Community-tested usability

🔄 Testing Process

  • Automated testing with axe-core
  • Manual testing with screen readers
  • Keyboard-only navigation testing
  • User testing with disabled community
  • Regular accessibility audits

  • NVDA (Windows) - Latest version
  • JAWS (Windows) - Version 2022+
  • VoiceOver (macOS/iOS) - Latest version
  • TalkBack (Android) - Latest version
  • Orca (Linux) - Version 40+

Voice Control

  • Dragon NaturallySpeaking
  • Voice Control (macOS)
  • Windows Speech Recognition
  • Voice Access (Android)

  • Enter/Space - Activate buttons and links

Alternative Input

  • Head pointer devices
  • Single-button input

Personalization Options

Use the accessibility panel (♿ button) in the bottom right to customize your experience.

🔆 High Contrast Mode

Switches to black background with white text and bright accent colors. Automatically activates if your system is set to high contrast mode.

Keyboard shortcut: Ctrl+Alt+C

🔍 Large Font Size

Increases all text by 20% for better readability. Works alongside your browser's zoom settings and respects your system font size preferences.

Keyboard shortcut: Ctrl+Alt+F

📖 Dyslexia-Friendly Font

Switches to OpenDyslexic font with increased letter spacing and word spacing. Designed to reduce reading errors and improve comprehension.

Keyboard shortcut: Ctrl+Alt+D

⏸️ Reduce Motion

Disables animations and transitions for users who experience vestibular disorders or prefer static content. Respects your system motion preferences.

Keyboard shortcut: Ctrl+Alt+M

Accessibility Feedback

Found an accessibility barrier? We want to hear from you and fix it quickly.

Report Accessibility Issues

If you encounter any accessibility barriers on our site, please let us know. We'll respond within 48 hours and work on fixes immediately.

Email: accessibility@disability-ai-collective.org

GitHub: File an accessibility issue

What to Include

  • URL of the page with the issue
  • Description of what you were trying to do
  • What assistive technology you're using
  • What browser and operating system
  • Any error messages you received

Alternative Formats

Response Time

We prioritize accessibility requests and aim to:

  • Acknowledge within: 24 hours
  • Critical fixes within: 48 hours
  • Full resolution within: 1 week
  • Updates every: 3 business days

Legal Compliance

  • ADA Title III - Americans with Disabilities Act
  • EN 301 549 - European accessibility standard
