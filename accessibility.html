Accessibility Statement

Accessibility isn't an afterthought—it's fundamental to who we are and how we build technology.

Our Commitment

As an experimental arts platform led by disability culture, accessibility guides every design decision.

Accessibility-First Design

We don't retrofit accessibility—we design with disabled users as primary stakeholders from day one.

Community Accountability

We're accountable to disabled communities first. Our practices are reviewed by disabled users representing diverse disabilities and assistive technologies.

Continuous Improvement

Digital accessibility is an ongoing practice, not a checklist. We regularly test with real users and update based on feedback.

Open Source

Our accessibility patterns and code are open source so other organizations can benefit from disability-led design practices.

WCAG 2.1 AA Compliance

We meet or exceed WCAG 2.1 Level AA standards with ongoing testing and community verification.

✅ What We Provide

  • 4.5:1 minimum color contrast (we exceed this)
  • Text resizes to 200% without losing functionality
  • All images have descriptive alt text
  • Keyboard navigation for all interactive elements
  • Skip links for efficient navigation
  • High contrast and dyslexia-friendly options
  • Screen reader optimized structure
  • 48px minimum touch targets on mobile

🔄 How We Test

  • Automated testing with accessibility tools
  • Manual testing with screen readers
  • Keyboard-only navigation testing
  • User testing with disabled community members
  • Regular accessibility audits
  • Real-world assistive technology testing
  • Color contrast verification
  • Mobile accessibility validation

Assistive Technology Support

Our site works with the assistive technologies you rely on.

Screen Readers

Tested and optimized for:

  • NVDA (Windows)
  • JAWS (Windows)
  • VoiceOver (macOS/iOS)
  • TalkBack (Android)
  • Orca (Linux)

Voice Control

  • Dragon NaturallySpeaking
  • Voice Control (macOS)
  • Windows Speech Recognition
  • Voice Access (Android)

Keyboard Shortcuts

  • Tab/Shift+Tab - Navigate interactive elements
  • Enter/Space - Activate buttons and links
  • Escape - Close menus and dialogs
  • Arrow keys - Navigate within components

Alternative Input

  • Switch navigation devices
  • Eye tracking systems
  • Head pointer devices
  • Single-button input

Accessibility Features

Look for the accessibility toggles in the bottom-left corner of your screen.

🔆 High Contrast Mode

Switches to enhanced contrast with bright colors for better visibility.

Automatically activates if your system uses high contrast mode.

📖 Dyslexia-Friendly Font

Switches to OpenDyslexic font with improved letter spacing for easier reading.

Designed to reduce reading errors and improve comprehension.
